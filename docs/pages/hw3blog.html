In this assignment, I had trouble with search. First, making the search work properly came with a whole set of problems. Personally, in my web development experience I always hide elements with css. However, the requirements wanted us to "completely" remove the elements not matching the query from the DOM, therefore I had to figure out how to do that. What I did to work through the issue was check if the substring of whe query was not in the text of the viewable twits. If true, I removed the element.

Another issue I had was actually creating the tweet programatically. I had to figure out how to create the individual parts of the twit (the author, content, icon). What I did was create individual functions to do each part of the creation. I then called the functions in a "main" function which actually submits the twit by appeding the combined sub-elements into one and submitting that HTMLElement to the DOM. Making the process modular really helped me in this assignment, especially when I had to inevitably debug the code.

I learned how to manipulate the DOM in new ways, how to programatically create an element in vanilla JS, and how to make modular client side javascript code. The most helpful resource during this specific assignment was the class lectures. I wasn't able to be at class due to being sick, so having recorded lectures helped when I came across issues that I couldn't solve and needed to implement some of the functionality according to the requirements.
